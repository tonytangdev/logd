## Codebase Patterns
- Spec: docs/superpowers/specs/2026-03-28-logd-design.md
- Plan: docs/superpowers/plans/2026-03-28-logd-implementation.md
- GitHub issues #1-#19 contain complete test files — copy them verbatim
- Quality checks: npm run format:check && npm run typecheck && npm test
- Biome for formatting/linting, vitest for tests
- vitest.config.ts must exclude `dist/**` — tsc outputs .js test files that vitest picks up
- CLI pattern: `src/cli/index.ts` creates Commander program + wires deps; `src/cli/commands/*.ts` export register functions
- CLI tests use `execSync` against built `dist/bin/logd.js` with `LOGD_DB_PATH` env var pointing to temp dir
- DI pattern: core services take repo/client interfaces as constructor params
- sqlite-vec virtual tables: embedding stored in separate decisions_vec table, joined by id
- Qwen3-Embedding: 1024-dim vectors, no prefix for documents, instruction prefix for queries
- Biome v2: uses `includes` not `ignore` in files config; run `npx biome migrate --write` if schema mismatch
- Biome v2: `biome check` = format+lint, `biome format` = format only
- MCP server: use `McpServer` from `@modelcontextprotocol/sdk/server/mcp.js` (high-level API), `StdioServerTransport` for stdio
- MCP tests: use `InMemoryTransport.createLinkedPair()` + `Client` from SDK for in-process tool invocation
- E2E tests: use async `exec` (promisified) NOT `execSync` — execSync blocks the event loop, deadlocking in-process mock HTTP servers
- sqlite-vec: load via `import * as sqliteVec from "sqlite-vec"; sqliteVec.load(db)`
- createDatabase(dbPath) is the single entry point for DB setup — returns a configured Database instance
- Use `node:` protocol for Node.js builtin imports (e.g. `from "node:fs"`) — Biome enforces this
- Map snake_case DB columns to camelCase TS properties via typed row interface + mapper function
- sqlite-vec queries: pass `new Float32Array(embedding)` for vector ops; distance col → score via `1 - distance`
- JSON array fields in DB: store via `JSON.stringify()`, parse via `JSON.parse()` on read, null check first
- decisions.project FK references projects(name), not projects(id) — Decision type uses project name directly
- Always run `npm run format` before `format:check` — biome enforces tab indentation on JSON too
- tsconfig has `types: ["node"]` — @types/node installed as devDep for Node.js globals (process, etc)
- Use `randomUUID()` from `node:crypto` instead of `uuid` package — uuid v13 is ESM-only, incompatible with CJS project
- updatedAt in service layer: use `Math.max(Date.now(), existing + 1)` to guarantee strictly-later timestamp in tests
- CLI tests needing Ollama: add `{ timeout: 30_000 }` to describe block; ensure qwen3-embedding:0.6b is pulled (not 8b — wrong dims)
- Commander repeatable flags: `.option("-a, --flag <val>", "desc", collectFn, [])` with `function collect(v, prev) { return [...prev, v]; }`
- vitest `describe` timeout: use `describe("name", { timeout: 30_000 }, () => {...})` — options as 2nd arg, fn as 3rd
