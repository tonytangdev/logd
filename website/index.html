# You mention a decision to your agent

> Use SQLite instead of Postgres for this project.
> We need zero-config deployment and a
> single-file database for the CLI tool.
# Your agent enriches and logs it via MCP

$ logd add "Use SQLite over Postgres" \
    -p backend \
    -c "Zero-config deploy, single-file DB" \
    -a "Postgres" -a "MySQL" \
    -t database -t infrastructure

# Months later, search by meaning

$ logd search "why not a hosted database?"

Use SQLite over Postgres | backend | 0.89 | a1b2c3...

Up and running in minutes

logd needs Ollama for local embeddings and an MCP server to connect your agent. Here's the full setup.

1

Install

Install logd and pull the embedding model into Ollama.

npm i -g @tonytangdev/logd ollama pull qwen3-embedding:0.6b
2

Connect

Start the MCP server so your agent can talk to logd.

logd serve
3

Decide & Recall

Mention decisions to your agent. Search the "why" months later.

Your agent does the work

Say "use SQLite" and your agent fills in the context, alternatives, tags, and reasoning. You decide. It documents.

Search by meaning

Not keyword matching. Ollama embeds every decision locally, so "why not a hosted DB?" finds "Use SQLite over Postgres."

Never leaves your machine

SQLite database, local embeddings, no cloud sync. Your architectural decisions stay on your disk, period.
